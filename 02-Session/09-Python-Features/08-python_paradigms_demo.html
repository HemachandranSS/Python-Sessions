Exploring Procedural vs Object-Oriented Programming

üìã Procedural Programming

Key Benefits:

  • Simple and straightforward approach
  • Easy to understand for beginners
  • Functions are reusable
  • Top-down problem solving
# Procedural Banking System accounts = {} def create_account(account_id, balance=0): accounts[account_id] = balance return f"Account {account_id} created" def deposit(account_id, amount): if account_id in accounts: accounts[account_id] += amount return f"${amount} deposited" return "Account not found" def get_balance(account_id): return accounts.get(account_id, 0)

üèóÔ∏è Object-Oriented Programming

Key Benefits:

  • Data encapsulation and security
  • Code reusability through inheritance
  • Better code organization
  • Polymorphism for flexibility
# Object-Oriented Banking System class BankAccount: def __init__(self, account_id, balance=0): self.__account_id = account_id self.__balance = balance def deposit(self, amount): self.__balance += amount return f"${amount} deposited" def get_balance(self): return self.__balance # Usage account = BankAccount("12345", 1000)
