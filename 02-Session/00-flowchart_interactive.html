Flow Chart Symbols and Meanings

START

Terminal/Oval

Purpose: Start and End points

Usage: Marks where the program begins and terminates

Code Example: Function start/end, program entry/exit

PROCESS

Process/Rectangle

Purpose: Processing steps

Usage: Calculations, assignments, function calls

Code Example: result = a + b

?

Decision/Diamond

Purpose: Decision points

Usage: Conditions that branch the flow

Code Example: if (x > 0)

I/O

Input-Output/Parallelogram

Purpose: Data input or output

Usage: User input, display results

Code Example: prompt(), alert()

A

Connector/Circle

Purpose: Connect flow parts

Usage: Link sections, avoid crossing lines

Code Example: Function calls, goto labels

ðŸŽ¯ Key Flow Chart Principles:

  • Single Entry/Exit: Each flowchart should have one start and one end point
  • Clear Direction: Arrows show the flow direction clearly
  • Simple Logic: Each decision symbol should have only yes/no or true/false outcomes
  • Readable Labels: Use clear, concise text in each symbol
  • Structured Flow: Avoid unnecessary complexity and crossing lines

ðŸŽ¨ Flow Chart Best Practices:

  • Use Standard Symbols: Stick to conventional shapes for clarity
  • Maintain Consistent Size: Keep symbols proportional and readable
  • Label Clearly: Use descriptive text inside each symbol
  • Show All Paths: Every decision should show both YES and NO paths
  • Avoid Crossing Lines: Use connectors to keep the chart clean
  • Test the Logic: Walk through the chart to verify it works correctly
