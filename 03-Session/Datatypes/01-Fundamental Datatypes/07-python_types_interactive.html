📊 Understanding bool() Function

The bool() function converts values to True or False based on specific rules for each data type:

🔢 Integer (int)

0 → False

Any non-zero → True

🔸 Float

0.0 → False

Any non-zero → True

🌊 Complex

0+0j → False

Any other value → True

📝 String (str)

Empty string "" → False

Any non-empty string → True

🔄 Type Casting Functions

🔢 Integer Conversion

Convert to int

int(123.987) → 123
int(True) → 1
int(False) → 0
int("10") → 10
int("10.5") → ValueError
int(10+5j) → TypeError

🔸 Float Conversion

Convert to float

float(10) → 10.0
float(True) → 1.0
float("10.5") → 10.5
float("ten") → ValueError
float(10+5j) → TypeError

🌊 Complex Conversion

Convert to complex

complex(10) → (10+0j)
complex(10, -2) → (10-2j)
complex(True, False) → (1+0j)
complex("10") → (10+0j)

📝 String Conversion

Convert to str

str(10) → '10'
str(10.5) → '10.5'
str(10+5j) → '(10+5j)'
str(True) → 'True'

🔐 Object Immutability & Identity

Fundamental data types in Python are immutable. When you create objects with the same value, Python often reuses the same memory location for efficiency.

💾 Memory Objects

💡 Key Concepts

🔒 Immutability

Once created, fundamental data type objects cannot be changed. Any modification creates a new object.

♻️ Object Reuse

Python reuses objects with identical values to save memory and improve performance.

🆔 Identity vs Equality

is checks if two variables point to the same object. == checks if values are equal.

🎯 Type Safety

Each conversion function has specific rules. Complex numbers cannot be converted to int or float.
