üìä Integer Data Type Overview

In Python, the int data type represents whole numbers (integral values). Unlike Python 2, Python 3 uses only the int type for all integers, including very large values.

            

a = 10
print(type(a))  # <class 'int'>

# Python 3 can handle arbitrarily large integers
big_number = 12345678901234567890
print(big_number)  # No overflow!

Key Points:

  • Python 3 unified integer representation (no separate long type)
  • Supports arbitrarily large integers (limited only by memory)
  • Four different literal representations: decimal, binary, octal, hexadecimal
  • All values are displayed in decimal form by default

üî¢ Number Systems

Python supports four different number systems for representing integers:

Decimal

Base 10

Default system

Digits: 0-9

a = 42

Binary

Base 2

Prefix: 0b or 0B

Digits: 0, 1

a = 0b101010

Octal

Base 8

Prefix: 0o or 0O

Digits: 0-7

a = 0o52

Hexadecimal

Base 16

Prefix: 0x or 0X

Digits: 0-9, A-F

a = 0x2A

üö´ Invalid Examples from Your Content (These cause errors):

  • a = 0B123 ‚ùå SyntaxError: invalid digit '2' in binary literal
  • a = b111 ‚ùå NameError: name 'b111' is not defined (missing 0b prefix)
  • a = 0o786 ‚ùå SyntaxError: invalid digit '8' in octal literal
  • a = 0XBeer ‚ùå SyntaxError: invalid digit 'r' in hexadecimal literal

‚úÖ Correct Versions and Key Points:

  • Binary: Only digits 0 and 1 allowed. Use 0b or 0B prefix.
  • Octal: Only digits 0-7 allowed. Use 0o or 0O prefix.
  • Hexadecimal: Digits 0-9 and A-F allowed (case insensitive). Use 0x or 0X prefix.
  • Important: PVM (Python Virtual Machine) always displays values in decimal form, regardless of how you enter them.

üìö Key Insights from Your Content:

  • Python 2 had separate long type, but Python 3 uses only int for all integers
  • You can specify literals in decimal, binary, octal, and hexadecimal forms
  • PVM always provides output in decimal form only
  • Base conversion functions (bin, oct, hex) work with any input base
  • The example a=10, b=0o10, c=0X10, d=0B10 shows same syntax but different values!
